# かおかくし。仕様メモ(Claude Code引き継ぎ用)

作成日: 2026-07-07 / 作成: Claude(claude.ai チャット)との対話で決定した内容のまとめ

## 1. このアプリの目的

Zoomのスクリーンショットや講座の集合写真をSNS(主にInstagramストーリー・Threads)に投稿する際、
**主催者(自分)以外の参加者・受講生の顔を簡単に隠す**ためのツール。

これまでCanvaで手作業(四角を描いて1人ずつぼかしを配置)していた処理を自動化する。

## 2. ターゲットと利用シーン

- 利用者: 講座主催者・コンサルタント本人と、その受講生(非エンジニア)
- シーンA: **Zoomスクショ** … ギャラリービューの参加者タイルを四角くぼかし、主催者タイルだけ見せる
- シーンB: **リアル講座の集合写真** … バラバラに写る受講生の顔だけ個別にぼかし、自分だけ見せる
- 環境: PCブラウザ + スマホブラウザの両対応(インストール不要のWebアプリ)

## 3. コアコンセプト(UXの肝)

**「隠す作業」ではなく「見せる人を選ぶ作業」にする。**

1. 画像をアップロード(D&D / タップ選択 / PCはCtrl+V貼り付け)
2. 顔を自動検出 → **全員に一括でぼかし**がかかる
3. 見せたい顔(主催者)だけタップ → ぼかし解除(再タップで戻る)
4. 保存 → そのままSNSへ

人数が多いほど手作業との差が開く(10人でもタップ1回)。

## 4. プロトタイプ(kaokakushi.html)の実装済み機能

- 画像入力: ドラッグ&ドロップ / ファイル選択 / クリップボード貼り付け(Ctrl+V / ⌘+V)
- 顔の自動検出: face-api.js の TinyFaceDetector(inputSize 512, threshold 0.4)
- 検出した顔全員に自動でぼかし → タップでON/OFF切り替え
- 手動で四角ぼかしを追加(ドラッグで範囲指定、右上✕で削除)… 検出漏れの保険
- 加工の種類: ぼかし(Gaussian) / モザイク(ピクセル化)の切り替え
- ぼかし範囲: 「顔まわり」/「大きめ」(Zoomタイル風に広めに覆う簡易版)
- 強さスライダー(4〜40)
- PNG保存(ガイド線は保存画像に含めない)
- **完全クライアントサイド処理**(画像はサーバーに送信しない)← 重要な設計方針
- EXIF回転の正規化(createImageBitmap / imageOrientation)
- スマホ対応: Pointer Events、タッチ操作、レスポンシブ
- ctx.filter非対応ブラウザ(古いSafari等)ではモザイクにフォールバック

## 5. 技術構成

- 単一HTMLファイル(HTML + CSS + Vanilla JS)。ビルド不要
- 顔検出: face-api.js 0.22.2(CDN読み込み) + TinyFaceDetectorモデル(CDNから取得)
- 描画: Canvas 2D。ぼかしは clip + ctx.filter、モザイクは縮小→拡大
- 座標はすべて元画像ピクセル基準。表示はCSSで縮小

### プロトタイプの既知の制約(Claude Codeで改善したい点)

- モデルの重み(weights)を外部CDNから取得している
  → **本番版はモデルファイルをローカル同梱**にしてオフライン・安定動作にする
- 環境によってはCDNがブロックされ自動検出が動かない(その場合は手動モードのみ)
- 「大きめ」モードは顔ボックスの拡大にすぎず、Zoomタイルの正確な矩形検出ではない

## 6. Claude Codeでの開発ロードマップ(優先順)

1. **【最優先】Zoomタイル自動吸着(Zoomモード)**:
   画像からタイルのグリッド線(黒い境界)を自動検出し、タイルの矩形を割り出す。
   ユーザーは**タイルをタップするだけ**で、そのタイル全体にピタッと綺麗な四角ぼかしがかかる。
   手描き不要になるので、スマホの指でもPCのマウスでも操作精度が変わらない。
   顔検出と組み合わせ「人がいるタイルは最初から全部ぼかし→主催者タイルだけタップで解除」が理想形。
   (参考: Canva加工画像のような、綺麗に四角く切り取られた仕上がり)
   ※既存のモザイクアプリにない、このアプリ最大の差別化ポイント。
2. **モデルのローカル同梱**: face-api.jsのweightsをプロジェクト内に置き、CDN依存をなくす
   (代替: @vladmandic/face-api や MediaPipe Face Detection への乗り換えも検討可)
3. **手動四角の微調整ハンドル**: 描いた四角の角・辺にハンドルを表示し、
   あとから位置・サイズをドラッグで調整できるようにする(Canvaと同じ操作感)。
   集合写真など、タイルがない不規則な画像での精度を補う
4. **GitHub Pagesで公開**: 自分専用URLを発行し、PC・スマホどちらからでも使えるようにする
5. **PWA化**: スマホのホーム画面に追加でき、オフラインでも動くようにする
6. その他アイデア: JPEG書き出し(容量削減)、ストーリー用9:16トリミング、
   目線バー・スタンプなどぼかし以外の隠し方、複数枚一括処理

## 7. 変えないでほしい設計方針

- **画像を外部に送信しない**(端末内処理)。受講生のプライバシー保護が大前提
- 操作は「アップ → タップ → 保存」の3ステップを超えない
- 非エンジニア・スマホユーザーが迷わないUI(日本語、ボタンは大きく)
- 「自動検出 + 手動追加」の二段構え(自動は横顔・小さい顔を見逃すことがあるため)
- 保存前に隠し漏れ確認を促す文言を残す
